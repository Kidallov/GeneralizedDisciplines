# Отчет по практической работе 1

## Тема

Базовая 2D-игра "Инопланетное вторжение" с использованием библиотеки `pygame`.

## Цель работы

Разработать игру, в которой игрок управляет космическим кораблем, стреляет по флоту инопланетян, теряет жизни при столкновениях и проходит уровни с увеличивающейся сложностью.

## Структура проекта

- `alien_invasion.py` - главный файл игры: запуск, игровой цикл, обработка событий, столкновения, переходы между уровнями.
- `settings.py` - статические и динамические настройки игры: размеры окна, скорости, лимиты, параметры сложности.
- `ship.py` - класс корабля игрока и логика движения вправо/влево.
- `bullet.py` - класс снаряда, движение вверх и отрисовка.
- `alien.py` - класс инопланетянина и проверка выхода флота за края экрана.
- `game_stats.py` - текущая статистика, жизни, счет, уровень, сохранение рекорда.
- `scoreboard.py` - отображение счета, рекорда, уровня и количества кораблей.
- `button.py` - кнопка запуска игры.
- `sound_effects.py` - процедурные звуковые эффекты выстрела, попадания и конца игры.
- `requirements.txt` - зависимость `pygame`.
- `README.md` - краткая инструкция по запуску и управлению.

## Выполнение заданий

1. Настройка окружения: добавлен файл `requirements.txt` с зависимостью `pygame>=2.5`. В проекте создано локальное окружение `.venv`, куда установлен `pygame`. При необходимости установку можно повторить командой `python3 -m pip install -r requirements.txt`.
2. Игровое окно: окно создается в `alien_invasion.py` через `pygame.display.set_mode`, размер и цвет фона задаются в `settings.py`.
3. Основной цикл: метод `run_game` содержит бесконечный цикл, где обрабатываются события, обновляются корабль, пули и пришельцы, затем перерисовывается экран.
4. Класс `Ship`: реализован в `ship.py`. Корабль появляется по центру нижней части экрана и перемещается стрелками влево/вправо без выхода за границы.
5. Стрельба: класс `Bullet` реализован в `bullet.py`. Пули появляются у носа корабля, движутся вверх и удаляются после выхода за верхнюю границу. Количество одновременных пуль ограничено настройкой.
6. Флот инопланетян: класс `Alien` реализован в `alien.py`. В `alien_invasion.py` создается несколько рядов пришельцев. Флот движется горизонтально, меняет направление у края окна и опускается вниз.
7. Конец игры и ограничения: при столкновении с пришельцем или достижении пришельцем нижней границы игрок теряет корабль. После потери всех кораблей игра завершается и появляется кнопка повторного запуска.
8. Организация кода: проект разделен на обязательные модули из задания и дополнительные вспомогательные модули для статистики, интерфейса и звука.
9. Дополнительный функционал: реализованы уровни, начисление очков, сохранение рекорда и звуковые эффекты.

## Описание игрового процесса

Игрок запускает игру, нажимает `P`, `Enter` или кнопку `PLAY`, затем управляет кораблем стрелками и стреляет пробелом. За каждого уничтоженного пришельца начисляются очки. Когда весь флот уничтожен, начинается новый уровень: скорость корабля, пуль и пришельцев увеличивается, а стоимость пришельца в очках возрастает.

## Использованные подходы ООП

Проект построен на нескольких классах с разделением ответственности. Главный класс `AlienInvasion` координирует игровой цикл, но не хранит всю логику внутри себя: движение корабля находится в `Ship`, поведение пули - в `Bullet`, поведение пришельца - в `Alien`, параметры игры - в `Settings`, а статистика - в `GameStats`. Такой подход упрощает чтение, тестирование и дальнейшее расширение игры.

## Проверка работоспособности

Проверена синтаксическая корректность Python-файлов с помощью компиляции. Также выполнен headless smoke-test с dummy SDL-драйверами: игра создала флот, выполнила выстрел и один цикл обновления объектов без ошибки.

## Вывод

В результате создана полноценная 2D-игра "Инопланетное вторжение", соответствующая требованиям практической работы: есть управление кораблем, стрельба, флот инопланетян, система жизней, завершение игры, уровни, счет и сохранение рекорда.
