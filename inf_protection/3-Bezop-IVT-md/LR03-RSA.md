## ИСТОРИЧЕСКИЕ СВЕДЕНИЯ

В современном мире *асимметричные криптосистемы* (иногда называемые криптосистемами с открытым ключом) решают следующие задачи:

1. Конфиденциальность информации в открытых каналах;
2. Цифровая подпись сообщений в открытых каналах.

**Аксиома Керкгоффса** [Шнайер,1999,с.42]:
Стойкость шифра должна определяться только секретностью ключа.

**Определение** (по [Шнайер,1999,с.42]):
*Криптосистема с открытым ключом* — это шифр, обладающий свойствами:
1. Вычислительно легко создать пару ключей (открытый и закрытый);
2. Вычислительно легко, зная открытый ключ, зашифровать сообщение;
3. Вычислительно легко, зная закрытый ключ, расшифровать сообщение;
4. Вычислительно сложно восстановить закрытый ключ, зная открытый ключ.

**Определение** [Молдовян,2006]:
*Криптосистема* — это совокупность обратимых преобразований, реализующих преобразования открытого текста в шифртекст.

---

## Шифры с открытым ключом

**Определение** (по [Шнайер,1999,с.19-20]):
*Криптосистема с открытым ключом* — это криптосистема, в которой для шифрования и дешифрования используются разные ключи.

**Свойства шифров с открытым ключом**:
1. *Симметричные криптосистемы* используют один ключ.
2. *Асимметричные криптосистемы* используют разные ключи.

**Определение** [Молдовян,2006]:
*Криптосистема с открытым ключом* — это криптосистема, в которой открытый ключ не позволяет вычислить закрытый ключ.

## ОДНОСТОРОННИЕ ФУНКЦИИ

> *Диффи и Хеллман ввели новую идею в криптографию — идею односторонней функции. Это была одна из тех идей, которые кажутся совершенно очевидными после того, как они уже высказаны, но до этого никто их не замечал.*
> *С.Сингх. Книга шифров*

**Определение** (по [Шнайер,1999,с.20]):
*Односторонняя функция* — это функция, вычисление которой легко, а обращение — трудно.

**Определение** (по [Шнайер,1999,с.22]):
*Односторонняя функция с "потайным ходом" (trapdoor function)* — это односторонняя функция, обращение которой становится легким при наличии секретной информации.

## КРИПТОСИСТЕМА RSA

### 1. Алгоритм генерации ключей

**Генерация ключей RSA** (по [Шнайер,1999,с.42]):

1. Выбираются два больших простых числа $p$ и $q$ (простые числа должны быть примерно одинаковой длины).
2. Вычисляется модуль $n = p \cdot q$.
3. Вычисляется функция Эйлера $\varphi(n) = (p-1)(q-1)$.
4. Выбирается открытый ключ $e$, такой что $1 < e < \varphi(n)$ и $\gcd(e, \varphi(n)) = 1$.
5. Вычисляется закрытый ключ $d$ из условия $e \cdot d \equiv 1 \pmod{\varphi(n)}$.

**Пара ключей**:
- Открытый ключ: $(e, n)$
- Закрытый ключ: $(d, n)$

### 2. Шифрование и дешифрование

**Шифрование** сообщения $m$ (где $0 < m < n$):
$$c = m^e \bmod n$$

**Дешифрование** шифртекста $c$:
$$m = c^d \bmod n$$

**Доказательство корректности**:

1. Если $\gcd(m, n) = 1$, то по теореме Эйлера:
   $$c^d = m^{ed} = m^{1 + k\varphi(n)} = m \cdot (m^{\varphi(n)})^k \equiv m \pmod n$$

2. Если $\gcd(m, n) > 1$, то $m$ кратно $p$ или $q$. В этом случае также можно показать, что $m^{ed} \equiv m \pmod n$.

## КРИПТОАНАЛИЗ RSA

### Выбор простых чисел

Для обеспечения стойкости RSA необходимо, чтобы простые числа $p$ и $q$ удовлетворяли определённым требованиям:

1. $(p-1, q-1)$ должны иметь большой общий делитель;
2. Простые множители $p-1$ и $q-1$ должны содержать большой простой множитель;
3. $p-1$ и $q-1$ должны иметь большой простой множитель;
4. $p+1$ и $q+1$ должны иметь большой простой множитель;
5. $(p-1)/2$ и $(q-1)/2$ должны быть простыми (безопасные простые числа).

**Пример получения безопасного простого числа в Maple**:
```
> safeprime(60871000);
                             60871367
```

### Выбор открытого ключа $e$

На практике часто используются значения $e = 3$, $e = 17$, $e = 2^{16}+1 = 65537$. Это позволяет ускорить шифрование.

### Вычисление закрытого ключа $d$

Закрытый ключ $d$ вычисляется как обратный элемент к $e$ по модулю $\varphi(n)$:
$$d = e^{-1} \bmod \varphi(n)$$

Это можно сделать с помощью расширенного алгоритма Евклида.

## КРИПТОСИСТЕМА ЭЛЬ-ГАМАЛЯ

**Алгоритм Эль-Гамаля** (по [Шнайер,1999,с.28]):

1. Выбирается большое простое число $p$ и первообразный корень $g$ по модулю $p$.
2. Выбирается секретный ключ $x$ ($1 < x < p-1$).
3. Вычисляется открытый ключ $y = g^x \bmod p$.

**Шифрование**:
Для сообщения $m$ выбирается случайное число $k$ ($1 < k < p-1$):
- $a = g^k \bmod p$
- $b = m \cdot y^k \bmod p$
- Шифртекст: $(a, b)$

**Дешифрование**:
$$m = b \cdot (a^x)^{-1} \bmod p$$

## КВАДРАТИЧНОЕ ШИФРОВАНИЕ

Если $p \equiv 3 \pmod 4$ и $q \equiv 3 \pmod 4$, то шифрование по схеме Рабина:
$$c = m^2 \bmod n$$

Для дешифрования используются формулы:
$$a_1 = c^{(p+1)/4} \bmod p,\quad b_1 = c^{(q+1)/4} \bmod q$$
$$a_2 = p - a_1,\quad b_2 = q - b_1$$

Затем с помощью китайской теоремы об остатках получаются четыре возможных корня.

## ПРИМЕРЫ РЕШЁННЫХ ЗАДАЧ

### Пример 1. Реализация RSA

**Условие**: Сгенерировать ключи RSA для $p=23$, $q=43$, $d=47$, зашифровать сообщение $m=205$.

**Решение**:

1. $n = 23 \cdot 43 = 989$
2. $\varphi(n) = (23-1)(43-1) = 22 \cdot 42 = 924$
3. Открытый ключ $e$ находим из условия $e \cdot d \equiv 1 \pmod{\varphi(n)}$:
   $e = 59$
4. Шифрование: $c = 205^{59} \bmod 989 = 818421$ (разбиваем на блоки: 818 и 421)
5. Дешифрование: $818^{47} \bmod 989 = 173$, $421^{47} \bmod 989 = 205$

---

### Пример 2. Протокол Диффи-Хеллмана

**Условие**: Реализовать обмен ключами по протоколу Диффи-Хеллмана с $p=23$, $a=5$, $b=7$.

**Решение**:

1. Выбирается общее простое число $p=23$ и первообразный корень $g$ (например, $g=5$).
2. Секретные ключи: $x=5$, $y=7$.
3. Открытые ключи: $A = g^x \bmod p = 5^5 \bmod 23 = 21$, $B = g^y \bmod p = 5^7 \bmod 23 = 17$.
4. Общий секрет: $K = B^x \bmod p = 17^5 \bmod 23 = 20$ (проверка: $A^y \bmod p = 21^7 \bmod 23 = 20$).

## ПРАКТИЧЕСКАЯ РЕАЛИЗАЦИЯ RSA

### Пример реализации на языке Lisp

```lisp
; Реализация криптосистемы RSA для цифровой подписи
;   p    - простое число
;   Text - исходное сообщение (Text < p-1)
; ---------------------------------
(DEFUN Crypto (LAMBDA (p Text)
   (SETQ a 5) ; a - открытый ключ абонента A
   (SETQ b 7) ; b - открытый ключ абонента B
   ; Вычисление Alpha - закрытого ключа A
   (SETQ Alpha (Modul a (Euler p 1) (- (Euler (Euler p 1) 1) 1)))
   ; Вычисление Betta - закрытого ключа B
   (SETQ Betta (Modul b (Euler p 1) (- (Euler (Euler p 1) 1) 1)))
   ; Шифрование
   (SETQ m1 (Modul Text p a))
   (SETQ m2 (Modul m1 p b))
   (SETQ m3 (Modul m2 p Alpha))
   (RETURN (Modul m3 p Betta))
))

; Функция быстрого возведения в степень
(DEFUN StepQuick (LAMBDA (n m)
   (COND ((ZEROP m) 1)
         ((ZEROP (REM m 2)) (StepQuick (* n n) (/ m 2)))
         (T (* n (StepQuick (* n n) (/ (- m 1) 2)))))
))

; Функция модульного возведения в степень
(DEFUN Modul (LAMBDA (n a m)
   (COND ((ZEROP m) 1)
         ((ZEROP (REM m 2)) (Modul (REM (* n n) a) a (/ m 2)))
         (T (REM (* n (Modul (REM (* n n) A) a (/ (- M 1) 2))) a)))
))
```
